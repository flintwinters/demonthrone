- Project state: browser proof of concept for a crisp Three.js/WebGL isometric tactics grid.
- Run locally by opening `index.html` or serving the directory with any static file server.
- TypeScript source lives in `ts/`; compiled browser modules are emitted to `src/` by `npm run build`.
- Reusable object-oriented domain classes for biome profiles, terrain kinds, safe zones, noise layers, height profiles, and character templates live in `ts/domain.ts`; feature modules should compose those classes rather than rebuilding parallel tables.
- Generic tile identity, cardinal adjacency, and Manhattan distance helpers live in `ts/grid.ts`; feature modules should import them instead of redefining tile math.
- Current scope is coordinate-unbounded height-aware Three.js voxel grid rendering, biome-driven integer-height terrain with height-tinted vertical side-face gradients, panning, wheel/pinch zooming, right-drag vertical/horizontal camera rotation, mouse hover tile highlighting, tile selection, two-character elevation-aware line of sight with living teammate/enemy sight blockers, configurable character sight/movement/attack-range/health parameters, random enemy generation, configurable enemy pursuit of the nearest teammate after every living-team turn whether or not enemies are visible, enemy range attacks against teammate health, one-turn gray sphere tombstones for destroyed teammates, shadowless teammate markers, vertical cone enemy rendering, biome-weighted Perlin-noise procedural boulder, passable biome-colored biome-weighted sight-reducing crossed flat triangle brush, teammate-pushable crates with simultaneous movement previews, toggleable teammate-bound crate enchantments with obstacle-aware pursuit, and obstacle-aware L1-limited simultaneous planned movement with highlighted start/end tiles and transparent destination unit previews.
- Performance state: terrain faces are batched in `ts/terrain-batch.ts`, boulder/brush props render through `InstancedMesh` in `ts/terrain-props.ts`, and renderer visibility results are cached across camera/hover redraws in `ts/renderer.ts`.
- Python backend lives in `backend/`, uses only the standard library, serves the static frontend, and exposes `/api/health` plus `/api/game/new`.
- Verification gate: run `npm run check`; ESLint enforces cyclomatic complexity and file length limits.
- Backend verification gate: run `python -m unittest discover`.
- UI direction: dense high-contrast gruvbox dark theme, minimal labels, no animations or transitions.
- Use TypeScript as much as possible. Try not to hand-edit JavaScript.
- Stop rm-ing __pycache__

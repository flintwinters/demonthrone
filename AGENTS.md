- Project state: browser proof of concept for a crisp Three.js/WebGL isometric tactics grid.
- Run locally by opening `index.html` or serving the directory with any static file server.
- TypeScript source lives in `ts/`; compiled browser modules are emitted to `src/` by `npm run build`.
- Reusable object-oriented domain classes for biome profiles, terrain kinds, safe zones, noise layers, height profiles, and typed entity templates live in `ts/domain.ts`; feature modules should compose those classes rather than rebuilding parallel tables.
- Entity contracts form a discriminated hierarchy in `ts/types.ts`: all entities have stable identity and archetype metadata, damageable characters and world objects share health, and teammate/enemy/pushable branches add only their own state. Add new archetypes through the matching template subclass rather than constructing ad hoc object literals.
- Generic tile identity, cardinal adjacency, and Manhattan distance helpers live in `ts/grid.ts`; feature modules should import them instead of redefining tile math.
- Current scope is coordinate-unbounded height-aware Three.js voxel grid rendering, high-contrast biome-driven integer-height terrain with height-tinted vertical side-face gradients, impassable flat Perlin-wet basin-filled water bodies at varied local-minimum elevations with tenfold line-of-sight traversal efficiency, passable basin-filled Perlin ice with the same sight multiplier and half-cost weighted movement, panning, unbounded positive wheel/pinch zooming, right-drag vertical/horizontal camera rotation, mouse hover tile highlighting, tile selection, health-bearing teammates/enemies/crates with compact overhead numeric health labels, turn-consuming teammate attacks against enemies and crates with entity-only range highlights and attacker-specific line of sight, two-character elevation-aware line of sight with living teammate/enemy sight blockers, configurable character sight/movement/attack-range/health parameters, coordinate-unbounded thresholded Perlin-local-maximum enemy and crate fields that materialize with newly traversed terrain and retain consumed origins, configurable enemy pursuit before each player planning phase, enemy range attacks during turn resolution without a same-phase move, one-turn gray sphere tombstones for destroyed teammates and enemies, shadowless teammate markers, vertical cone enemy rendering, sparse biome-weighted Perlin-noise procedural boulders, passable biome-colored biome-weighted high-LOS-cost crossed flat triangle foliage with stable coordinate-varied heights, teammate-pushable crates with simultaneous movement previews, turn-consuming crate-first enchantment binding to an explicitly selected teammate or enchanted chain link with downstream chain dispelling, unified click action precedence and click-again/teammate/Escape cancellation, and obstacle-aware weighted simultaneous planned movement with reciprocal exponential uphill/downhill slope costs and highlighted start/end tiles and transparent destination unit previews.
- Performance state: terrain faces are batched in `ts/terrain-batch.ts`, boulder/brush props render through `InstancedMesh` in `ts/terrain-props.ts`, and renderer visibility results are cached across camera/hover redraws in `ts/renderer.ts`.
- Visibility fields use cached octant shadowcasting with fixed angular bins, slope-scaled weighted range cost, and vertical terrain/boulder/character horizons in quadratic work; attacker-specific entity LOS retains the exact native 3D supercover ray traversal.
- Movement and attack overlays are cached per-unit fields: movement uses one obstacle-aware weighted flood fill, while attacks reuse the 3D shadowcasting geometry with attack range as the budget and unit traversal cost.
- Python backend lives in `backend/`, uses only the standard library, tolerates clients disconnecting during static responses, serves the static frontend, and exposes `/api/health` plus `/api/game/new`.
- Verification gate: run `npm run check`; ESLint enforces cyclomatic complexity and file length limits.
- Backend verification gate: run `python -m unittest discover`.
- UI direction: dense high-contrast gruvbox dark theme, minimal labels, no animations or transitions; foliage is deliberately dimmer than other scene objects, remains visibly green, and uses explicit geometry normals to participate in strong scene lighting rather than glowing unlit or rendering black.
- The lower-left selection status contains concise context-sensitive interaction guidance or selected entity/terrain info, with no heading or supplemental statistics; internal brush terrain is labeled `foliage`.
- Use TypeScript as much as possible. Try not to hand-edit JavaScript.
- Stop rm-ing __pycache__
